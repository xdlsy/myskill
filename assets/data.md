# DATA文档通用模板

## 基础信息

本DATA文档模板用于记录系统模块的核心数据相关信息，包含数据结构、数据表设计、配置定义三大模块，每个字段均附带详细注释，明确字段含义、用途、取值范围及业务逻辑，为开发、测试、运维人员提供统一的数据参考标准，保障数据设计、开发、维护的一致性和规范性。

## 数据结构定义

定义模块核心业务数据的结构体（实体类），涵盖所有业务相关字段，每个字段均添加详细注释，明确字段用途、取值范围、默认值及业务约束，适配主流开发语言。

### 核心业务实体结构（Java示例）

```Java
/**
 * 核心业务实体（示例：用户信息实体）
 * 说明：用于存储系统用户的基础信息、权限关联信息，是模块核心数据载体
 */
public class UserEntity {
    /**
     * 用户唯一主键ID
     * 类型：Long
     * 取值范围：正整数，自增序列（1-99999999）
     * 用途：唯一标识一个用户，作为数据库主键，关联其他业务表
     */
    private Long userId;

    /**
     * 用户名
     * 类型：String
     * 取值范围：2-20位字符，由字母、数字、下划线组成，不允许特殊字符
     * 用途：用户登录账号，唯一不可重复，用于身份识别
     */
    private String username;

    /**
     * 用户手机号
     * 类型：String
     * 取值范围：11位纯数字，符合中国大陆手机号格式（13/14/15/17/18/19开头）
     * 用途：用户绑定手机号，用于登录验证、短信通知、找回密码
     */
    private String phone;

    /**
     * 用户状态
     * 类型：Integer
     * 取值范围：0-禁用，1-正常，2-锁定，3-注销
     * 用途：控制用户账号的可用状态，默认值为1（正常）
     */
    private Integer status;

    /**
     * 创建时间
     * 类型：Date
     * 取值范围：标准时间格式（yyyy-MM-dd HH:mm:ss）
     * 用途：记录用户账号的创建时间，由系统自动生成，不可手动修改
     */
    private Date createTime;

    /**
     * 最后登录时间
     * 类型：Date
     * 取值范围：标准时间格式（yyyy-MM-dd HH:mm:ss），可为null
     * 用途：记录用户最后一次登录系统的时间，每次登录时自动更新
     */
    private Date lastLoginTime;

    /**
     * 扩展字段
     * 类型：String
     * 取值范围：最大500位字符，可为null
     * 用途：存储用户相关的额外信息（如备注、自定义配置等），根据业务需求灵活使用
     */
    private String ext;

    //  getter/setter 方法（省略，实际开发中需补充）
}
```

### 核心业务实体结构（Go示例）

```Go
/**
 * 核心业务实体（示例：用户信息实体）
 * 说明：用于存储系统用户的基础信息、权限关联信息，是模块核心数据载体
 */
type UserEntity struct {
    UserId        int64     `json:"userId" gorm:"primaryKey;comment:'用户唯一主键ID，正整数自增（1-99999999），唯一标识用户，作为数据库主键'"`
    Username      string    `json:"username" gorm:"size:20;not null;unique;comment:'用户名，2-20位字符（字母、数字、下划线），唯一不可重复，用于登录身份识别'"`
    Phone         string    `json:"phone" gorm:"size:11;not null;unique;comment:'用户手机号，11位纯数字（13/14/15/17/18/19开头），用于登录验证、短信通知'"`
    Status        int       `json:"status" gorm:"not null;default:1;comment:'用户状态，0-禁用，1-正常，2-锁定，3-注销，默认值为1'"`
    CreateTime    time.Time `json:"createTime" gorm:"not null;comment:'用户创建时间，标准时间格式（yyyy-MM-dd HH:mm:ss），系统自动生成'"`
    LastLoginTime time.Time `json:"lastLoginTime" gorm:"comment:'最后登录时间，标准时间格式（yyyy-MM-dd HH:mm:ss），可为null，登录时自动更新'"`
    Ext           string    `json:"ext" gorm:"size:500;comment:'扩展字段，最大500位字符，存储用户额外信息，灵活适配业务需求'"`
}
```

## 数据表定义

定义模块所有数据表的详细信息，包含表基本信息、字段详情、表关联关系，每个字段均附带注释，明确字段含义、存储规则、约束条件，适配主流关系型数据库。

### 核心业务数据表（示例：用户信息表）

| 表名       | sys_user（系统用户表） | 表注释：存储系统所有用户的基础信息、状态信息及关联信息，是模块核心数据表，关联角色表、权限表 |
| ---------- | ---------------------- | ------------------------------------------------------------ |
| 数据库类型 | MySQL 8.0              | 字符集：utf8mb4，排序规则：utf8mb4_general_ci                |

#### 表字段详情

| 字段名          | 字段类型     | 是否主键 | 是否非空 | 默认值            | 字段注释                                                     |
| --------------- | ------------ | -------- | -------- | ----------------- | ------------------------------------------------------------ |
| user_id         | BIGINT(20)   | 是       | 是       | AUTO_INCREMENT    | 用户唯一主键ID，自增序列（1-99999999），唯一标识一个用户，作为表主键，关联其他业务表（如角色关联表） |
| username        | VARCHAR(20)  | 否       | 是       | 无                | 用户名，2-20位字符，由字母、数字、下划线组成，不允许特殊字符，唯一不可重复，用于用户登录账号和身份识别 |
| phone           | VARCHAR(11)  | 否       | 是       | 无                | 用户手机号，11位纯数字，符合中国大陆手机号格式（13/14/15/17/18/19开头），唯一不可重复，用于登录验证、短信通知、找回密码 |
| status          | TINYINT(1)   | 否       | 是       | 1                 | 用户状态，取值范围：0-禁用（账号不可登录）、1-正常（账号可正常使用）、2-锁定（账号临时不可登录）、3-注销（账号永久失效），默认值为1 |
| create_time     | DATETIME     | 否       | 是       | CURRENT_TIMESTAMP | 用户账号创建时间，标准时间格式（yyyy-MM-dd HH:mm:ss），由数据库自动生成，不可手动修改，用于追溯账号创建记录 |
| last_login_time | DATETIME     | 否       | 否       | NULL              | 用户最后一次登录系统的时间，标准时间格式（yyyy-MM-dd HH:mm:ss），用户每次登录时自动更新，用于统计用户活跃度 |
| ext             | VARCHAR(500) | 否       | 否       | NULL              | 扩展字段，最大500位字符，用于存储用户相关的额外信息（如用户备注、自定义配置、临时标识等），根据实际业务需求灵活使用 |

#### 表关联说明

- 关联表1：sys_user_role（用户角色关联表），关联字段：user_id（当前表）→ user_id（关联表），用于绑定用户与角色的关联关系，实现权限控制
- 关联表2：sys_operation_log（操作日志表），关联字段：user_id（当前表）→ operator_id（关联表），用于记录用户的操作行为日志

## 配置定义信息

定义模块所有配置项的详细信息，包含配置基本信息、配置字段详情，每个配置项均附带注释，明确配置用途、取值范围、默认值及修改影响，适配不同环境（开发/测试/生产）的配置需求。

### 配置基本信息

| 配置文件名称 | application.yml（或application.properties） | 说明：模块核心配置文件，存储所有业务相关配置项，按环境区分配置（dev/test/prod） |
| ------------ | ------------------------------------------- | ------------------------------------------------------------ |
| 配置加载方式 | 系统启动时自动加载                          | 说明：配置修改后需重启系统生效（或配置动态刷新机制）         |

### 核心配置项详情

| 配置项名称       | 配置key                    | 配置类型 | 默认值                              | 适用环境 | 配置注释                                                     |
| ---------------- | -------------------------- | -------- | ----------------------------------- | -------- | ------------------------------------------------------------ |
| 数据库地址       | spring.datasource.url      | String   | jdbc:mysql://localhost:3306/db_name | 所有环境 | 模块数据库连接地址，db_name替换为实际数据库名称，修改后需重启系统生效，确保数据库连接正常 |
| 数据库用户名     | spring.datasource.username | String   | root                                | 所有环境 | 数据库登录用户名，开发/测试环境可使用默认值，生产环境需使用专用账号，保障数据安全 |
| 数据库密码       | spring.datasource.password | String   | 123456                              | 所有环境 | 数据库登录密码，开发/测试环境可使用默认值，生产环境需设置复杂密码，定期更换，避免泄露 |
| 用户登录超时时间 | system.user.login-timeout  | Integer  | 3600（秒）                          | 所有环境 | 用户登录后的会话超时时间，单位为秒，超时后用户需重新登录，可根据业务需求调整（如生产环境设置为1800秒） |
| 分页最大条数     | system.page.max-size       | Integer  | 50                                  | 所有环境 | 模块所有分页接口的最大每页条数，防止一次性查询过多数据导致系统性能下降，取值范围1-100 |
| 日志打印级别     | logging.level.com.example  | String   | dev:DEBUG;prod:INFO                 | 所有环境 | 模块日志打印级别，开发环境设为DEBUG（便于调试），生产环境设为INFO（减少日志冗余），避免影响系统性能 |

### 配置注意事项

- 所有配置项的key命名遵循驼峰规范，语义清晰，避免重复
- 生产环境配置需隐藏敏感信息（如数据库密码、密钥等），建议使用配置中心管理
- 配置项默认值仅适用于开发/测试环境，生产环境需根据实际部署情况调整
- 修改配置后，需确认配置格式正确（如yaml格式缩进、数据类型匹配），避免系统启动失败

## 通用规范

1. 数据结构、数据表、配置项的命名遵循驼峰规范（数据表、字段名可用下划线分隔），语义清晰，避免使用模糊、无意义的名称
2. 所有字段/配置项必须添加注释，注释需详细、准确，明确用途、取值范围及业务约束，便于后续维护
3. 数据表主键统一使用自增Long类型，外键关联需明确关联关系，避免冗余数据
4. 配置项需区分环境，开发、测试、生产环境的配置值需合理区分，保障环境一致性
5. 数据结构、数据表、配置定义需随业务迭代同步更新，确保文档与实际开发一致
