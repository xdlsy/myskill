# Code Insight - 代码分析洞察

## 概述

此技能用于对任意代码项目进行全面分析，帮助用户快速理解项目结构、技术栈、架构设计和模块实现、业务流程等。分析结果以 Markdown 报告形式呈现。

先充分理解当前项目背景，然后**一次只提一个问题**来逐步加深理解。等你确信已经基本了解项目信息后，再开始后续动作。

<HARD-GATE>
在你提交分析计划并获得用户批准之前，严禁调用任何skill、编写任何代码、搭建任何项目脚手架，或落地任何实现。无论项目看似多么简单，该要求均适用于所有项目。
</HARD-GATE>

## Checklist

你必须为下面的每一个项目创建一个任务，并依次执行：

1. **探索项目上下文** — 检查目录内现有的文件和文档

2. **模块拆分** — 根据目录结构拆分模块，每拆分完一个模块，都要征得用户同意再继续

3. **模块分析** — 分析每个模块，并生成分析报告，存放在模块根目录的docs目录（不存在则创建）内

4. ****

## The Process

**探索项目上下文**

* 使用 Glob 工具探索项目根目录，识别：

  - 依赖管理文件（package.json、requirements.txt、go.mod 等）

  - 配置文件（tsconfig.json、.eslintrc、webpack.config.js 等）

  - 入口文件（index.js、main.py、main.go 等）

  - 文档文件（README.md、docs/ 目录等）


* 对项目进行简单总结，随时通过提问与用户澄清。

**模块拆分**

* 根据目录结构将项目拆分为逻辑模块：

  - 按功能域拆分（api/、models/、services/、utils/ 等）

  - 按技术栈拆分（frontend/、backend/、shared/ 等）

  - 每拆分完一个模块，呈现给用户并确认后再继续

  - 识别模块间的依赖关系
    - 最小粒度识别模块


  - 不放过任何一个存在源代码的目录

  - 如果不确定如何拆分，请通过提问与用户澄清


**模块分析**

* 对每个模块进行深入分析：

  - 主要文件和数据结构，生成data.md，**尽量遵循模版：**assets/data.md。

  - 对外暴露的接口，生成api.md，**尽量遵循模版：**assets/api.md。

  - 依赖的其他模块与接口调用关系，生成dependency.md，**尽量遵循模版：**assets/dependency.md。


* 每个模块完成分析后，需要征得用户同意后再分析下一个模块。
* 将分析报告保存至 `/**/<模块名>/docs/`。
* **尽量遵循assets目录中定义的模版，当模版中的某些内容不存在时，也没必要硬写**。

## Key Principles

- **边分析边确认**：每完成一个模块的分析，与用户确认后再继续。
- **生成完整数据报告**：每一个模块分析完毕后，都要生成完成的分析报告，包括：api.md、data.md、dependency.md。

* **尽量遵循模版**：尽量遵循assets中定义的模版，如果模版中的某些内容不适用或不存在，也可以跳过，
